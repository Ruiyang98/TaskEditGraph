# 任务编辑界面 Spec

## Why
需要一个可视化的任务编辑界面来管理仿真实体的任务流。用户可以通过图形化方式查看任务之间的连接关系，直观地了解任务状态，并能够方便地对任务进行增删改操作以及设置任务切换条件。

## What Changes
- 新增基于 Qt5 的任务编辑图形界面组件
- 实现任务节点（TaskNode）的绘制与状态显示
- 实现任务节点之间的连线（Connection）绘制
- 实现右键上下文菜单，支持添加、修改、删除任务
- 实现任务切换条件配置：执行指定时间后切换、顺序执行切换、指定时间开始切换
- 支持不同颜色表示任务状态（等待、运行中、已完成、失败等）

## Impact
- 新增模块：TaskEditor, TaskNode, Connection, TaskModel, TaskConditionDialog
- 依赖 Qt5 Widgets / GraphicsView 框架
- 无现有代码变更（新项目）

## ADDED Requirements

### Requirement: 任务节点可视化
The system SHALL 在图形视图中绘制任务节点，每个节点显示任务名称和当前状态，并用不同颜色边框/背景表示状态。

#### Scenario: 显示多个任务节点
- **GIVEN** 存在任务列表 [任务1, 任务2, 任务3]
- **WHEN** 打开任务编辑界面
- **THEN** 视图中水平排列显示三个任务节点，节点之间用箭头连线表示执行顺序

### Requirement: 任务状态颜色区分
The system SHALL 使用不同颜色表示任务状态：
- 等待中（Pending）：灰色
- 运行中（Running）：绿色
- 已完成（Completed）：蓝色
- 失败（Failed）：红色

### Requirement: 右键上下文菜单
The system SHALL 在任务节点上右键点击时弹出上下文菜单，提供以下操作：
- 添加任务（在当前节点后插入新任务）
- 修改任务（编辑任务名称和属性）
- 删除任务（移除当前任务并断开连接）

### Requirement: 任务切换条件配置
The system SHALL 支持为每个任务设置切换到下一条任务的条件，条件类型包括：
- 执行指定时间后切换：设置持续时间（秒），任务运行达到该时间后自动切换
- 顺序执行切换：当前任务完成后立即切换
- 指定时间开始切换：设置绝对时间点（HH:MM:SS），到达该时间后切换

#### Scenario: 配置时间条件
- **GIVEN** 用户右键点击任务节点并选择"设置切换条件"
- **WHEN** 在弹出的对话框中选择"执行指定时间后切换"并输入 10 秒
- **THEN** 该任务的切换条件被保存，连线旁显示条件标识

### Requirement: 任务数据模型
The system SHALL 提供任务数据模型（TaskModel）管理任务列表，支持增删改查操作，并维护任务间的连接关系。
